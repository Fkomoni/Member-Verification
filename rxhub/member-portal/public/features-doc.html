1. Overview

The LeadwayHMO RxHub is a production-grade PBM (Pharmacy Benefit Management) Member Self-Service Platform that empowers Leadway Health enrollees to manage their medication lifecycle, track health readings, and submit change requests — all through a structured approval workflow integrated with the Prognosis PBM system.

The platform eliminates manual PBM operations, improves medication adherence, and provides full audit visibility for administrators.

2. Member Portal

The member-facing web portal allows enrollees to self-serve their medication needs, health tracking, and profile management.

Live URL: https://rxhub-member.onrender.com

2.1 Authentication

Primary Login

Enrollee ID + Phone Number validated against the live Prognosis API in real-time.

OTP Fallback

If phone doesn't match, OTP is sent to the registered phone via Termii SMS gateway.

Session Security

JWT-based authentication with 5-hour session timeout and automatic token refresh.

2.2 Dashboard

  • Welcome message with member's real name, scheme, and diagnosis from Prognosis
  • Stats cards: active medication count, pending requests, unread alerts
  • Upcoming refill alerts with next refill dates and urgency indicators
  • Quick action buttons for common tasks

2.3 Medications Management

Real medication data synced from the Prognosis PharmacyDelivery/GetPbmMedication API on every login.

View All Medications

Drug name, dosage, frequency, days supply, next refill date, last refill date.

Request Refill

One-click refill request per medication, submitted for admin approval.

Request All Refills

Bulk refill request for all active medications at once.

Add Medication

Request new medication with prescription upload (PDF/JPG/PNG). Goes through approval.

Remove Medication

Request removal with reason. Submitted through the Change Request Engine.

2.4 Health Readings

Members can log and track three types of health readings over time with clinical trend analysis.

Reading TypeFieldsClassifications
Blood PressureSystolic, Diastolic (mmHg)Normal, Elevated, High Stage 1, High Stage 2
Blood GlucoseLevel (mg/dL) + Context (Fasting/Random/Post-meal)Normal, Pre-diabetic, Diabetic Range
CholesterolTotal, HDL, LDL, Triglycerides (mg/dL)Desirable, Borderline High, High

Trend Tracking

IMPROVING / STABLE / WORSENING analysis comparing recent readings with historical data.

Change Indicators

Shows exact change vs previous reading (e.g. "-7/-6 mmHg") with color-coded arrows.

CSV Download

Export all readings with dates, values, and classifications for a chosen period.

Date Range Filter

Filter history by type and date range for focused analysis.

3. Admin Portal

The management portal for PBM administrators to review requests, manage resources, and monitor platform activity.

Live URL: https://rxhub-admin.onrender.com

3.1 Request Management

Grouped by Member

All requests organized by member ID. Click to expand and see all pending requests for one enrollee.

Inline Actions

Approve, Reject, or Modify each request directly with comment — no page navigation needed.

Bulk Actions

"Approve All" or "Reject All" pending requests for a member in one click.

Status Filtering

Filter by Pending, Approved, Rejected, or Modified across all members.

3.2 Audit Logs

  • Complete audit trail of every action: who, what, when, before vs after state
  • Filter by request ID or actor ID
  • Paginated for large datasets

3.3 Analytics Dashboard

  • Total active members
  • Members with activity in last 30 days
  • Request stats: pending, approved, rejected, modified counts

3.4 Resource Management

  • Create, edit, delete newsletters, health tips, drug alerts, scarcity alerts
  • Tag resources by diagnosis for targeted delivery
  • Publish/unpublish control

4. Prognosis PBM Integration

The platform integrates with the Leadway Health Prognosis system via authenticated REST APIs.

EndpointPurpose
POST /ApiUsers/LoginAuthenticate and obtain bearer token
GET /EnrolleeProfile/GetEnrolleeBioDataByEnrolleeIDFetch member profile (name, phone, scheme, employer)
GET /PharmacyDelivery/GetPbmMedicationFetch medications, refill dates, delivery info, diagnosis
  • Token caching with 55-minute TTL, automatic refresh on 401
  • Retry logic (3 attempts with exponential backoff)
  • Full sync logging to sync_logs table
  • Fallback to local DB validation when Prognosis is unreachable
